# v0.4.5 — Guard Truth Source Fix

这是一次门禁可信度修复版，重点解决 post / orchestrator / CI 结果不一致的问题。

## 核心修复

| 模块 | 修复方向 |
|------|----------|
| Guard Registry | post / orchestrator / CI 使用同一套 guard 真相源 |
| WARNING 汇总 | 所有 WARNING 结构化写入 guard_summary.json |
| FTS5 Healthcheck | 检测 invalid fts5 file format，并尝试 rebuild / fallback |
| scene_delta | 从关键词推进改为叙事证据推进 |
| scene_causality | 支持身体代价、环境代价、关系代价、物件后果 |
| continuity | 从词重叠改为承接证据 |
| anti_ai | 统一正则与句式检测入口 |
| path_resolver | 降低 novels_root / slug / 卷目录强耦合 |
| title policy | 标题变化只记录，不擅自改写用户标题 |

## 为什么重要？

v0.4.5 之前，门禁系统存在"多个入口多个结果"的问题。
这会导致 post 日志里已经出现 WARNING，但 orchestrator 仍返回 0 WARNING。
v0.4.5 的目标是让引擎拥有一个唯一的真相来源。

## 新增文件

- `scripts/guard_registry.py` — 统一 guard 注册和执行
- `scripts/guard_result.py` — GuardFinding / GuardResult / GuardSummary 数据结构
- `scripts/anti_ai_patterns.py` — anti_ai 统一规则库
- `scripts/consequence_lexicon.py` — 身体代价、物件代价、社会代价、规则代价词库
- `scripts/fts_health.py` — FTS5 检查、重建、fallback 检索
- `scripts/path_resolver.py` — 路径解析，支持自定义目录模板
- `scripts/bridge_evidence_guard.py` — 连续性承接证据报告
