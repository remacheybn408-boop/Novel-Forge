# v0.4.0 Release Notes

**版本**: v0.4.0
**代号**: Human-Grade Revision Suite (拟人审稿质量套件)
**发布日期**: 2026-05-24
**基于**: v0.3.1-qgp

---

## 概述

v0.4.0 Human-Grade Revision Suite 在 v0.3.1-qgp 版本基础上，新增 7 个拟人审稿质量门禁，从审稿痕迹、具体锚点、场景因果、对白自然度、句式变化、投稿合规和最终投稿报告七个维度检查稿件质量。

**核心目标**: 不是检测 AI 率，而是让稿件读起来更像人类写作。

---

## 新增功能: 7 个拟人审稿门禁

### 1. Editor Revision Guard (审稿痕迹检查)
检测"初稿感" — 过度解释、冗余修饰、解释链过完整等问题。成熟稿件通常有适当的省略和留白空间。
- 报告: `editor_revision_report.json`
- 级别: WARNING only

### 2. Concrete Anchor Guard (具体锚点守卫)
检测文字中是否锚定具体物件、身体动作和场景细节。抽象心理描写需要具体锚点支撑。
- 报告: `concrete_anchor_report.json`
- 级别: WARNING only

### 3. Scene Causality Guard (场景因果链)
检测场景是否具备完整的因果链 (CARCRH: 原因→行动→阻力→代价→结果→钩子)。
- 报告: `scene_causality_report.json`
- 级别: WARNING only

### 4. Dialogue Naturalness Guard (对白自然度)
检测对话是否有打断、未完成句、称呼变化和动作节拍，避免书面化完整轮替。
- 报告: `dialogue_naturalness_report.json`
- 级别: WARNING only

### 5. Style Variation Guard (句式变化守卫)
检测段落开头重复率、抽象词密度和句长变化度。
- 报告: `style_variation_report.json`
- 级别: WARNING only

### 6. Compliance Selfcheck Guard (投稿合规自查)
检查内容是否触碰平台投稿红线。**唯一可 BLOCK 入库的门禁。**
- 报告: `compliance_selfcheck_report.json`
- 级别: 可 BLOCK (唯一)

### 7. Final Submission Report (最终投稿报告)
汇总所有门禁结果，给出投稿建议和优先级修改列表。
- 报告: `final_submission_report.json`
- 级别: 纯报告

## 升级指南 (从 v0.3.1-qgp)

### 配置文件
`config.example.json` 新增 `human_grade_revision` 配置段。将以下内容合并到你的 `config.json`:

```json
"human_grade_revision": {
  "enabled": true,
  "version": "v0.4.0",
  "mode": "warning_first",
  "quality_guards_warning_only": true,
  "compliance_guard_can_block": true,
  "editor_revision_guard": { "enabled": true, "max_over_explained_ratio": 0.28, "min_revision_texture_score": 0.45 },
  "concrete_anchor_guard": { "enabled": true, "window_chars": 500, "min_anchor_density": 0.22, "require_object_anchor": true, "require_body_action_anchor": true, "require_scene_anchor": true },
  "scene_causality_guard": { "enabled": true, "require_cause": true, "require_action": true, "require_resistance": true, "require_cost": true, "require_result": true, "require_hook": false },
  "dialogue_naturalness_guard": { "enabled": true, "min_dialogue_variation_score": 0.18, "prefer_interruption": true, "prefer_unfinished_sentence": true, "prefer_action_beats": true },
  "style_variation_guard": { "enabled": true, "max_repeated_opening_ratio": 0.25, "max_overused_abstract_words": 12, "min_sentence_length_variance": 0.35 },
  "compliance_selfcheck_guard": { "enabled": true, "block_high_risk": true, "warn_medium_risk": true },
  "final_submission_report": { "enabled": true, "top_revision_tasks": 5 }
}
```

## Breaking Changes

**无破坏性变更。**

- 所有新门禁默认均为 WARNING only 模式
- 合规自查虽然支持 BLOCK，但默认配置下高风险才阻断
- 通过 `config.json` 中的 `"enabled": false` 可关闭整个套件
- 单个门禁也可通过各自的 `"enabled": false` 独立关闭
- 已有 `config.json` 无需修改即可继续使用（新门禁会因缺少配置而安全跳过）

## 已知限制

1. **合规自查基于关键词匹配** — 可能产生误报，需要人工判断边界案例
2. **场景因果检测基于启发式规则** — 对非线性叙事和非常规结构的支持有限
3. **具体锚点检测依赖分词准确性** — 中文分词在特定语境下可能不准确
4. **对白自然度检测对古风/文言对话的适应性有限** — 与 classical_register_guard 互补
5. **所有门禁均为本地执行** — 不依赖外部 API，因此识别能力受限于规则库完整度
6. **7 个门禁串行执行** — 对长章节 (5000+ 字) 可能增加若干秒处理时间
